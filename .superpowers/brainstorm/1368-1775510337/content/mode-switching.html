Trade-offs

A: Two Views B: Unified C: Toggle Overlay
Pipeline complexity High — must generate two layouts per course Medium — one layout, lecture data inline Medium — one layout, lecture data as separate layer
UX clarity Clear separation, but student may miss one mode Everything visible, but can feel cluttered Clean by default, extra context on demand
Content model Two component trees per course Single tree, mixed block types Single tree, lecture blocks conditionally rendered
Progress tracking Separate per view? Shared? Gets confusing One progress path One progress path, overlay doesn't affect it
