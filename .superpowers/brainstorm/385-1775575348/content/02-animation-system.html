Backwards Compatibility

Existing string animations stay untouched.
The current { "type": "animation", "variant": "kmp" } format continues working. The new format adds "renderer" + "data" fields. AnimationBlock detects which format is used and routes accordingly.

StringRenderer = existing StringMatchAnimation (KMP, BM, RK). No changes.
New: ArrayRenderer, GraphRenderer, TableRenderer — all use Motion for smooth animations.
