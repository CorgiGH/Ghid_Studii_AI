🎓 Lecture context

What happens when you run a program?

Learn

When you double-click an executable or type a command, the OS creates a process — a living instance of that program with its own memory, files, and execution state.

Key insight: a process isn't always running on the CPU. Most of its life is spent waiting.

📖 Definition

Process: An instance of a program in execution — includes code (text section), activity state (PC, registers), stack, data section, and heap.

If you open the same program twice, how many processes exist? Are they the same or different?

Click to reveal ▼
Two separate processes. Each has its own memory, PID, and state. They share code on disk but are fully independent in memory. If one crashes, the other survives.

Process States

Learn

A process is always in exactly one state:

  • New — Being created
  • Ready — Waiting for CPU
  • Running — Executing instructions
  • Blocked — Waiting for I/O
  • Terminated — Done
EXAM

⚠ Professor's emphasis

"This WILL be on the exam. Draw the complete state diagram with all transitions. Students always confuse Blocked and Ready."

📈 Interactive Diagram — click any state
New: Memory allocated, PCB initialized. Not yet in ready queue.
Ready: Has everything. Waiting for scheduler to give CPU time.
Running: Actively executing. Only 1 per CPU core.
Blocked: Waiting for I/O. Goes to Ready (not Running) when done.
Terminated: Done. PCB stays until parent reads exit status.

A process is reading from disk. What state? When done, does it go straight to Running?

Click to reveal ▼
It's Blocked. When I/O completes it moves to Ready, not Running. Must wait for the scheduler. Most common exam mistake.

Process Control Block (PCB)

Learn

Every process has a PCB — the OS's "ID card" for that process. It stores everything needed to pause, resume, and manage the process.

Context Switching

When the CPU switches from one process to another, it does a context switch: save current process state to PCB, load next process state from its PCB.

This is pure overhead — typically 1-10µs. No useful work gets done during the switch.

EXAM

⚠ Exam pattern

"Given processes P1, P2, P3 with these arrival times and burst times, trace the context switches under FCFS scheduling." — Practice this.

💡 Think about it

Why must the program counter be saved in the PCB? What happens if it isn't?

Click to reveal ▼
Without saving the PC, a resumed process wouldn't know where it left off. It would restart from the beginning or crash. The PC is the "bookmark" of execution.

Key Takeaways

✓ A process is a running instance of a program with its own memory and state.

✓ Processes cycle through 5 states: New → Ready → Running → Blocked/Terminated.

✓ The PCB stores everything the OS needs to manage a process.

✓ Context switches save/restore PCB state — necessary but pure overhead.

✓ Blocked → Running is not allowed. Must go through Ready.
