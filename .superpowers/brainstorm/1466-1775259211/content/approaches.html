Navigation & Visual Design Approaches

Three directions for improving navigation flow and visual polish

A

Polish & Streamline (Evolutionary)

Keep the current sidebar + tabs layout but refine everything:

  • Sticky course-to-course navigation (prev/next buttons at bottom of each course)
  • Breadcrumb trail: Home → OS → Course 3
  • Animated tab transitions, better sidebar with progress rings
  • Quick-switch dropdown to jump between subjects without going Home
  • Visual refresh: better typography, spacing, card design

Pros

  • Least disruption — improves what exists
  • Faster to implement

Cons

  • Fundamental layout stays the same
  • Still relies on sidebar for course navigation
B

Unified Content Browser (Redesign)

Replace sidebar + tabs with a single unified navigation system:

  • Subject switcher always visible in top bar (dropdown or pill selector)
  • Content type (Courses/Seminars/Labs) as a secondary nav row, not tabs
  • Course list as expandable cards in the main content area, not a sidebar
  • Keyboard shortcuts: arrow keys to navigate, number keys to jump
  • Modern card-based design with progress indicators built in

Pros

  • Feels modern and seamless
  • No sidebar = more content space
  • Every level of navigation is always accessible

Cons

  • Bigger redesign effort
  • Need to rethink mobile layout too
C

Hybrid: Keep Sidebar, Redesign Everything Else

Sidebar stays for in-subject navigation, but redesign the shell around it:

  • Persistent top bar with subject switcher + content type selector
  • Redesigned sidebar: collapsible groups, progress bars, active state animations
  • Prev/next course navigation in content area
  • Breadcrumbs for orientation
  • Smooth transitions between views

Pros

  • Best of both: sidebar familiarity + modern shell
  • Moderate effort

Cons

  • Sidebar still takes horizontal space
